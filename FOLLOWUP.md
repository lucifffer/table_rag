### 7. 如果文档章节层级解析不对：假设这套系统解析后发现不少文档的章节层级不准确，例如标题、条款、段落、表格归属或跨页内容关系识别错，导致后续检索和问答效果受影响。请说明你会如何定位问题、判断影响范围、设计修复方案，并验证修复是否有效。也可以说明短期交付时如何止损，长期如何把这类问题做得更稳定。

根因分析：分析每种错误类型的根本原因，例如：

训练一个专门的文档结构解析模型，基于大量的标注数据进行训练

定性验证：人工检查修复后的解析结果，确保其符合预期
端到端验证：测试修复后的检索和问答效果，确保整体效果得到提升
回归测试：运行完整的回归测试，确保修复没有引入新的问题

### 8. 如果系统效果不稳定，请结合你提交的系统，选一个你认为最可能影响效果的问题，说明你会如何定义评价标准、收集 bad case、定位原因、尝试优化，并判断优化是否真的有效。

最可能影响效果的问题：表格内容的语义理解和检索匹配问题。这是扫描版 PDF RAG 系统中最常见也是最难解决的问题之一。表格通常包含结构化的信息，但传统的文本分块和向量检索方法很难捕捉到表格的语义结构和行列关系，导致表格问题的准确率远低于纯文本问题。

定义评价标准：
表格识别准确率：正确识别的表格数量 / 总表格数量
表格结构准确率：正确识别的行列数量 / 总行列数量
表格内容准确率：正确识别的单元格内容 / 总单元格数量
表格检索准确率：检索结果中相关表格的数量 / 总检索结果数量
表格问答准确率：正确回答的表格问题数量 / 总表格问题数量

收集 bad case 的方法：
设计一个专门的表格问题测试集，包含不同类型的表格问题，例如：
查找特定单元格的值
比较不同行或列的值
统计满足条件的行或列数量
查找最大值、最小值或平均值
收集用户在实际使用中遇到的表格问题和错误答案
对每个 bad case 进行详细记录，包括：
原始问题
系统返回的答案
正确答案
检索到的相关片段

错误原因分析
定位原因的方法：
解析阶段错误：表格识别不准确、结构错误或内容错误
分块阶段错误：表格被错误地分割成多个块，或者与周围的文本混合在一起
嵌入阶段错误：表格内容的向量表示不能准确捕捉其语义信息
检索阶段错误：检索算法不能有效地匹配表格内容和用户问题
生成阶段错误：大模型不能正确理解表格的结构和内容，生成错误的答案
尝试优化的方法：
表格解析优化：使用更先进的表格识别模型，或针对特定场景进行微调
表格表示优化：将表格转换为多种表示形式，包括：
纯文本格式（如 Markdown 表格）
结构化格式（如 JSON）
自然语言描述格式
表格分块优化：根据表格的结构和内容进行合理分块，保持表格的完整性
表格检索优化：
实现专门的表格检索器，结合关键词检索和语义检索
使用表格的元数据（如表头、行标题、列标题）进行检索增强
对表格的不同部分赋予不同的权重
结构生成优化：
在 prompt 中明确说明表格的结构和内容
引导大模型按照表格的行列关系进行推理
使用Agent技术提高答案的准确性

判断优化是否真的有效的方法：
定量评估：在测试集上计算优化前后的各项评价指标
A/B 测试：将优化前后的系统同时部署，让用户进行对比测试
用户反馈：收集用户对优化后系统的评价和反馈
bad case 分析：检查优化后原来的 bad case 是否得到了解决，以及是否引入了新的 bad case
